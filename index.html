Day 29 — CSS Display

Visual examples of display: block, display: inline and display: inline-block.

Quick reference

  • block — starts on a new line and stretches to the container width by default.
  • inline — flows inside a line of text; width and height properties are ignored.
  • inline-block — flows inline but accepts width and height like a block-level element.

Resize the browser to observe responsive behaviour. Use devtools to toggle display values live.

Notes & best practices

  1. Prefer semantic markup (<section>, <article>, <header>, <main>) for clarity and screen-reader friendliness.
  2. For horizontal alignment with flexible content, consider inline-block, Flexbox, or CSS Grid instead of forcing display changes on non-semantic elements.
  3. Test how margins and padding interact with different display values across viewports.
